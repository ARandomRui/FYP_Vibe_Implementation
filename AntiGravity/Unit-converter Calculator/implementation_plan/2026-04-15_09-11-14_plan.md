# Calculator with Unit Conversion & ANS History

## Problem Description
The goal is to build a calculator web application that supports standard mathematical operations and unit conversions (e.g., Miles to Kilometers, Fahrenheit to Celsius). It must feature an `ANS` operator representing the result of the previous equation. 

A unique UI state requirement is to display a history of equations interactively: 
- If the new equation incorporates `ANS` (e.g., `ANS + 5`), it is appended to the current display history. 
- If it does not use `ANS`, the display is cleared to show only the new equation, ensuring unrelated calculations start on a clean slate.

## Proposed Changes

### Configuration & Architecture
We will set up a modern frontend application (Vite + React, or Vanilla JS modules depending on your choice). The architecture will be strictly modular to decouple state management from the UI and logic.

### User Interface / Component Tree
#### [NEW] `index.html` / App Component
Main layout encompassing a history panel, a current input display panel, unit conversion controls, and the main numerical/operator keypad.

### State Management
#### [NEW] `stateManager.js` (or React hook)
A module to handle calculation state logic:
- `currentInputStr`: The ongoing user input.
- `historyList`: Array of past equations relevant to the current sequence.
- `lastAnswer`: Caching the float value of `ANS`.
- `shouldClearOnNextInput`: Flag managing whether an unrelated new calculation triggers a screen wipe.

### Calculator Logic
#### [NEW] `mathEvaluator.js`
A robust function to evaluate string-based mathematical expressions, swapping `ANS` keyword with the value of `lastAnswer` safely before computation.

#### [NEW] `unitConversions.js`
Pure functions establishing mappings and formulas for requested units.

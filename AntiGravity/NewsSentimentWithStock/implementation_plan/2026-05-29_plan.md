# Stock Sentiment Visualizer

This plan outlines the architecture and tasks required to build a web application that allows users to select a stock and view a visualization comparing recent stock price data with the sentiment analysis of recent news articles about that stock.

## User Review Required

## Proposed Architecture & Security Measures

- **Backend-for-Frontend (BFF)**: The frontend will never communicate directly with the stock/news APIs. It will request data from our own backend API routes, which will securely hold the API keys and perform the external requests.
- **Input Validation**: All user inputs (e.g., stock ticker symbols) will be strictly validated and sanitized to prevent injection attacks.
- **Content Security Policy (CSP)**: A strict CSP will be implemented to mitigate XSS.
- **Secure Configuration**: All secrets will be loaded from a `.env` file. We will include a `.env.example`.

## Proposed Changes

### Configuration and Setup
- Set up a new Next.js project with a focus on modern design aesthetics (vibrant colors, smooth micro-animations).
- Create `.env.example` defining required API keys.
- Install dependencies for charting and sentiment analysis, strictly validated by the `scan_dependencies` tool first.

### Backend (BFF API Routes)
- `api/stock`: Endpoint to fetch historical/recent stock prices for a given ticker.
- `api/news`: Endpoint to fetch recent news for a given ticker, perform sentiment analysis on the headlines/content, and return the aggregated sentiment scores with timestamps.

### Frontend
- **Landing/Search Page**: A beautifully designed search bar with micro-animations allowing the user to select or type a stock ticker.
- **Dashboard**: A responsive layout featuring:
  - A combined chart showing the stock price line and a sentiment trend line/bar.
  - A feed of recent news articles color-coded by their sentiment (e.g., green for positive, red for negative).

# Migrate News Source to Finnhub API

We will swap out the `yahoo-finance2` news scraper for the **Finnhub** free API. Finnhub's `companyNews` endpoint allows us to pull historical news for a specific stock ticker up to 1 year back on the free tier, perfectly matching our 30-day stock price window.

### Thought Process

**The core technical challenge**: Replacing a keyless scraper with an authenticated API without exposing secrets to the frontend, and mapping a completely different data schema into our existing frontend layout so we don't have to rewrite `Dashboard.tsx`.
**Potential edge cases**: 
- **Timezone and Date Formatting**: Finnhub returns dates as Unix timestamps (seconds). If we just pass this to the frontend, it will break our recent fix. We must intercept the Unix timestamp in the BFF (`route.ts`) and cast it to an ISO string *before* sending it to the client.
- **API Rate Limits**: The free tier of Finnhub limits requests (60 calls/minute). We need to implement proper error handling in the BFF to gracefully catch `429 Too Many Requests`.
- **Missing Data**: Some articles may lack summaries or thumbnails. We must fall back gracefully.
**Impact on existing system architecture**: 
- The frontend (`Dashboard.tsx`) remains completely untouched if we format the payload correctly on the backend.
- The backend (`/api/news/route.ts`) will drop `yahoo-finance2` and use native `fetch()`.
- An environment variable (`FINNHUB_API_KEY`) becomes a hard requirement.

## Proposed Changes

### Backend APIs
- Replace `YahooFinance` import with a standard `fetch` call to `https://finnhub.io/api/v1/company-news`.
- Read the API key using `process.env.FINNHUB_API_KEY`.
- Calculate dates for the `from` and `to` query parameters (last 30 days).
- Map the Finnhub response properties (`headline`, `summary`, `datetime`, `url`, `source`) to the structure expected by our frontend (`title`, `summary`, `providerPublishTime`, `link`, `publisher`).
- Run `vader-sentiment` on the `headline` and `summary` to retain our sentiment scoring.
- Convert the Finnhub `datetime` (Unix timestamp) into an ISO string so the frontend doesn't crash on date parsing.

### Environment Configuration
- Document the new `FINNHUB_API_KEY` requirement for future developers in `.env.example`.

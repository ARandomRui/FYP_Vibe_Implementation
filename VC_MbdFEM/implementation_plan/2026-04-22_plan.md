# FreeCAD MbD-FEM Integration Workbench

## Goal Description
Develop a custom FreeCAD workbench ("workspace") module that integrates Multibody Dynamics (MbD) and the Finite Element Method (FEM). This workbench will provide a functional GUI in FreeCAD and the core structural files to implement an explicit weak-coupling orchestrator, allowing kinematic loads to be translated into structural FEM stresses.

## User Review Required
> [!IMPORTANT]
> **One-Way vs. Two-Way Coupling:** Based on your previous prototype planning, I will implement **One-way Coupling** for MVP (Kinematics -> FEM loads, but structural deflection does not feed back to the kinematics). Do you approve?

## Proposed Changes

### Workbench Structure
We will create a standard FreeCAD module structure under `VC_MbdFEM/`:

#### [NEW] `Init.py`
Empty file to designate the directory as a FreeCAD/Python module.

#### [NEW] `InitGui.py`
FreeCAD Workbench class definition (`class MbdFemWorkbench`), which registers our custom toolbar and command icons inside the FreeCAD UX.

#### [NEW] `MbDFEM_Commands.py`
Creates FreeCAD command classes (e.g., `CmdMbdFemSolve`) that users click to trigger the orchestration.

#### [NEW] `mbd_solver.py`
A mathematical python engine to solve non-linear multi-body equations. Initial MVP will calculate basic 1-Degree-of-Freedom trajectory steps.

#### [NEW] `fem_translator.py`
Translates position/acceleration arrays into dynamic FreeCAD `ConstraintForce` objects acting on a specified target 3D CAD mesh.

#### [NEW] `mbdfem_orchestrator.py`
The master simulation loop. It links `mbd_solver.py` and `fem_translator.py` and triggers FreeCAD's native `ObjectsFem.makeSolverCalculix` iteratively in the background to calculate finite element solutions per time-step.

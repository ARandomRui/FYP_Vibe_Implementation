# Convert Modal Overlay to Full-Page Project View

This plan outlines the steps to change the current modal-based project detail view into a full-page "sub-site" view, providing more space for markdown content and chat logs.

## Background
Currently, when a user clicks on a project card, a modal overlay appears with a constrained width (`1000px`) and height (`90vh`). The user feels this is too small for the amount of content (plans, test cases, chat logs). 

## Proposed Changes

We will adopt a Single Page Application (SPA) routing approach. Instead of a modal, we will hide the main dashboard and display the project details as a full-page view within the same `index.html`. 

### 1. `index.html`
*   **[MODIFY]** `index.html`
    *   Change the `<div class="detail-overlay hidden" id="detail-overlay">` structure.
    *   Remove the `.detail-modal` wrapper.
    *   Introduce a new `.project-view-container` that acts as the new sub-site page.
    *   Redesign the header within this view: add a prominent "← Back to Dashboard" button, and restructure the title and agent badges to look like a full page header.

### 2. `css/style.css`
*   **[MODIFY]** `css/style.css`
    *   Remove modal-specific styles (`position: fixed`, `max-width`, `max-height`, `backdrop-filter` for overlay).
    *   Add styles for `.project-view-container` to take up the full width of the `app-container`.
    *   Style the new "Back to Dashboard" button.
    *   Expand the layout of `.detail-content` and `.tab-content-container` to utilize the newly available screen real estate (e.g., making chat logs wider and deeper).
    *   Ensure smooth CSS transitions when swapping between the dashboard view and the project view.

### 3. `js/app.js`
*   **[MODIFY]** `js/app.js`
    *   Update DOM element references (e.g., replace `modalOverlay` with `projectViewContainer`).
    *   Update `openProjectDetails()` to:
        1. Add `.hidden` to the main `.app-header` and `.dashboard`.
        2. Remove `.hidden` from `.project-view-container`.
        3. Scroll the window to the top (`window.scrollTo(0, 0)`).
    *   Rename `closeModal()` to `closeProjectDetails()` and update it to:
        1. Hide `.project-view-container`.
        2. Show the main header and dashboard.
    *   Remove logic that disables body scrolling (`document.body.style.overflow = 'hidden'`), as the new view is just a standard page that should allow scrolling.

## User Review Required

> [!IMPORTANT]
> **Design Choice (SPA vs Multi-page):** This plan proposes an SPA approach (hiding the dashboard and showing the detail view in the same `index.html` file). This is much cleaner and faster since all data is already loaded in `app.js`. 
> 
> Are you okay with keeping it all in `index.html` (functioning like a separate page visually), or do you strictly prefer creating a completely separate `project.html` file?

## Verification Plan
### Manual Verification
1. Click on a project card. Verify that the dashboard disappears and the project details expand to fill the entire page.
2. Verify that the new header looks distinct and prominent.
3. Verify that scrolling works naturally on long markdown files and chat logs.
4. Click "Back to Dashboard" and verify it cleanly returns to the main grid.

# Implementation Plan: Dynamic developer_notes.md Summary Redirection

## Overview
Redirect the showcase summary tab to point to a future `developer_notes.md` file in the project folders instead of local mock strings or `summary.md`. Ensure that if the file does not exist yet, the summary content renders as a clean empty state rather than breaking or throwing an HTTP fetch error.

## Technical Details
- **Location**: `ReviewWebsite`
- **Files Modified**:
  - `js/app.js`: 
    - Update `projectsData` summary paths to reference `developer_notes.md` for both projects.
    - Update `loadMarkdownContent` to intercept 404 errors (file not found) specifically for `developer_notes.md` and display a clean "Summary notes are empty" placeholder.
  - `css/style.css`: Add styles for `.empty-state-text` to ensure empty summaries look polished.

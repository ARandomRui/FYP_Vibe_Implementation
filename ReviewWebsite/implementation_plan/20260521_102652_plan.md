# Implementation Plan: Clarifying Chatbot Actions in Showcase

## Overview
Improve the chat log styling so that actions like `*Edited relevant file*` and `*Checked command status*` are clearly attributed to the chatbot (e.g., Antigravity or Codex) rather than looking like generic system logs or actions taken by the user.

## Technical Details
- **Location**: `ReviewWebsite`
- **Files Modified**:
  - `js/app.js`: Update parser and timeline renderer to tag action logs as agent-specific actions (`agent-action`) and attribute them to the active agent.
  - `css/style.css`: Add styles for agent action logs using the agent's custom accent colors and icons.

## Parsing & Attribute logic
1. When parsing `*Action*` blocks (lines starting and ending with `*`), classify them as `sender: 'agent-action'` instead of `system`.
2. Attribute the action to the current agent in context (e.g., `agent: agentName`).
3. In the renderer:
   - Instead of a generic gray gear icon, render these action logs using the agent's icon (rocket for Antigravity, code for Codex).
   - Style them with the agent's theme colors.
   - Format the text as `[Agent Name] [Action]` (e.g., `Antigravity: Edited relevant file` or `Antigravity accepted command...`).
